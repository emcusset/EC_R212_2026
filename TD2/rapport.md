Rapport — TD2
Points clés
1. Respect du HTML
Le CSS ne doit pas corriger le HTML mais révéler sa structure. J’ai compris qu’un bon HTML est déjà lisible sans style, et que le CSS sert uniquement à améliorer l’affichage sans modifier l’organisation du contenu.
2. CSS minimal et lisibilité
J’ai appris à écrire un CSS simple pour rendre une page lisible : choix d’une bonne typographie, ajout d’un interligne et limitation de la largeur de texte. Peu de lignes suffisent pour obtenir un rendu propre et confortable.
3. Layout avec Flexbox
J’ai utilisé Flexbox pour organiser les éléments (navigation, cartes) de manière fluide. Cela permet d’avoir un affichage adaptable sans casser le DOM et sans utiliser de valeurs fixes fragiles.
Utilisation de l'IA
Démarche
J’ai utilisé ChatGPT pour vérifier mes choix CSS, comprendre les bonnes pratiques et reformuler mes explications de manière plus claire et structurée.
Apports et limites
L’IA m’a aidé à mieux comprendre la notion de CSS maintenable et à éviter les erreurs comme les sélecteurs trop complexes ou les valeurs fixes. Cependant, certaines propositions étaient trop compliquées par rapport aux consignes du TD, j’ai donc dû simplifier.
